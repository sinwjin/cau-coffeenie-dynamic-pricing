# 310관 커피니 동적가격제 시뮬레이터

중앙대학교 다빈치 창의학점제 프로젝트용 Streamlit 웹앱입니다.

프로젝트 주제는 **중앙대학교 310관 커피니 아메리카노 동적가격제 시뮬레이터**입니다. 메뉴는 **아메리카노** 하나로 고정하고, 설문 응답 데이터를 기반으로 후보 가격별 구매확률과 예상매출을 비교합니다.

## 프로젝트 구조

```text
app.py
requirements.txt
README.md
sample_data.csv
.gitignore
```

`sample_data.csv`는 설문 파일이 없어도 Streamlit Cloud에서 바로 실행되도록 넣은 예시 데이터입니다. 앱은 업로드 파일이 없으면 이 파일을 사용하고, 이 파일을 읽지 못하는 경우에도 코드 내부 예시 데이터 생성 함수로 다시 실행됩니다.

## 모델 원리

사용 모델은 **Logistic Regression**입니다.

입력 변수:

- 요일
- 시간대
- 시험기간 여부
- 불쾌지수 단계
- 가격

출력값:

- 구매확률

설문 응답 변환:

- `구매하겠다` → `1`
- `구매하지 않겠다` → `0`

예상매출 공식:

```text
예상매출 = 가격 × 구매확률 × 예상 고객 수
```

발표용 시뮬레이션에서는 예상 고객 수 기본값을 `1`로 두고, 확장 가능성을 위해 앱 화면에서 예상 고객 수를 입력할 수 있게 했습니다.

## 가격 후보

아메리카노 가격 후보는 다음 7개입니다.

- 1600원
- 1800원
- 2000원
- 2200원
- 2400원
- 2600원
- 2800원

## 불쾌지수 단계

날씨 변수는 **불쾌지수 단계**로 사용합니다.

불쾌지수는 기온과 습도를 바탕으로 사람이 느끼는 불쾌감 정도를 나타내는 지표입니다. 다음 네 단계를 사용합니다.

| 불쾌지수 구간 | 단계 |
| --- | --- |
| 68 미만 | 낮음 |
| 68 이상 75 미만 | 보통 |
| 75 이상 80 미만 | 높음 |
| 80 이상 | 매우 높음 |

## 설문 데이터 형식

앱은 두 가지 형식의 설문 데이터를 처리할 수 있습니다.

### 1. Google Form식 Wide Format

응답자 1명이 한 행이고, 상황별 문항이 열로 들어오는 형식입니다.

예시:

| 이름 | 이메일 | 상황 1 | 상황 2 | 상황 3 |
| --- | --- | --- | --- | --- |
| 홍길동 | test@example.com | 구매하겠다 | 구매하지 않겠다 | 구매하겠다 |

앱은 `상황 1`부터 `상황 20`까지의 열을 찾아 `app.py` 내부의 `SITUATION_CONDITIONS`와 연결합니다. 이후 모델 학습용 long format으로 변환합니다.

개인정보로 보이는 열은 자동 제외합니다.

- 이름
- 성명
- email
- 이메일
- 전화
- 연락처
- phone
- tel
- 휴대폰

### 2. Long Format

이미 학습용으로 정리된 데이터도 업로드할 수 있습니다.

필수 열:

- 요일
- 시간대
- 시험기간 여부
- 불쾌지수 단계
- 가격
- 구매여부

`구매여부` 열에는 `구매하겠다`, `구매하지 않겠다`, `1`, `0` 값을 사용할 수 있습니다.
